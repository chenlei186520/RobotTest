# 日志文件说明

## 目录结构

```
logs/
├── 2026-01-14/          # 按日期组织的文件夹
│   ├── 2026-01-14_10-30-25.txt
│   ├── 2026-01-14_14-20-15.txt
│   └── ...
├── 2026-01-15/
│   └── ...
└── README.md
```

## 日志格式

- **文件夹命名**：`YYYY-MM-DD`（例如：2026-01-14）
- **日志文件命名**：`YYYY-MM-DD_HH-MM-SS.txt`（例如：2026-01-14_14-30-25.txt）
- **日志内容格式**：`YYYY-MM-DD HH:MM:SS [LEVEL] 消息内容`

## 日志内容

所有控制台打印信息（print语句）都会：
1. 自动添加时间戳
2. 保存到对应的日志文件中
3. 同时显示在控制台（保持原有格式）

## 日志轮转

- 单个日志文件最大：10MB
- 保留备份文件数：5个
- 超过大小限制时自动轮转

## 注意事项

- 日志文件使用UTF-8编码
- 日志文件按日期自动组织
- 每次启动应用会创建新的日志文件
- 控制台输出保持原有格式（不添加时间戳）
- 日志文件中的每条记录都包含时间戳
